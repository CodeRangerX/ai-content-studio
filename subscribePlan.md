## 二、点数方案设计

### 2.1 核心思路

**点数定义**: 1 点数 ≈ 1 次内容生成

**方案特点**:
- 适合轻度用户，不用订阅
- 点数永久有效（或 1 年有效期）
- 订阅用户仍然享受无限生成
- 点数用户与订阅用户可共存

### 2.2 点数包定价方案

#### 方案 A：保守定价（推荐）

| 点数包 | 价格 | 单价 | 适合人群 |
|--------|------|------|----------|
| 入门包 100 点 | $2.99 | ~$0.03/点 | 体验用户 |
| 标准包 300 点 | $6.99 | ~$0.023/点 | 轻度用户 |
| 进阶包 600 点 | $11.99 | ~$0.02/点 | 中度用户 |
| 专业包 1200 点 | $19.99 | ~$0.017/点 | 重度用户 |

**与订阅对比**:
- Pro 月付 $9 = 无限生成，适合重度用户
- 标准包 $6.99 = 300 次生成，适合轻度/不确定用户

#### 方案 B：激进定价（更便宜）

| 点数包 | 价格 | 单价 | 适合人群 |
|--------|------|------|----------|
| 入门包 150 点 | $2.99 | ~$0.02/点 | 体验用户 |
| 标准包 500 点 | $6.99 | ~$0.014/点 | 轻度用户 |
| 进阶包 1000 点 | $11.99 | ~$0.012/点 | 中度用户 |
| 专业包 2500 点 | $19.99 | ~$0.008/点 | 重度用户 |

### 2.3 点数规则

1. **有效期**: 永久有效（更友好）或 1 年有效期
2. **优先级**: 点数优先于订阅扣减（如果同时有）
3. **退款**: 点数一经购买不支持退款
4. **赠送**: 可设置注册赠送 10-20 点作为体验

### 2.4 用户场景对比

| 用户类型 | 月使用量 | 推荐方案 | 月成本 |
|----------|----------|----------|--------|
| 轻度用户 | 10-30 次 | 点数包 | $2.99-$6.99/数月 |
| 中度用户 | 30-100 次 | 点数包或订阅 | $6.99-$11.99/月 |
| 重度用户 | 100+ 次 | Pro 订阅 | $9/月 |
| 不确定用户 | 不确定 | 点数包 | 先买点数试用 |

### 3.2 点数消耗逻辑

```typescript
// 生成内容时检查
async function generateContent(userId: string, prompt: string) {
  const user = await getUser(userId);
  
  // 1. 检查是否有 Pro 订阅
  if (user.hasProSubscription) {
    return generateWithAI(prompt); // 无限生成
  }
  
  // 2. 检查点数余额
  if (user.credits.balance < 1) {
    throw new Error('INSUFFICIENT_CREDITS');
  }
  
  // 3. 扣减点数
  await deductCredit(userId, 1);
  
  // 4. 生成内容
  return generateWithAI(prompt);
}
```

## 四、推荐方案

### 最终推荐：方案 A + 微调

| 点数包 | 点数 | 价格 | 单价 | 有效期 |
|--------|------|------|------|--------|
| 入门包 | 100 | **$2.99** | $0.0299/点 | 永久 |
| 标准包 | 300 | **$6.99** | $0.0233/点 | 永久 |
| 进阶包 | 800 | **$14.99** | $0.0187/点 | 永久 |
| 专业包 | 2000 | **$29.99** | $0.015/点 | 永久 |

**理由**:
1. 定价有梯度，大包更划算
2. 最大包 $29.99，比订阅年付更有吸引力
3. 永久有效降低用户决策门槛
4. 保持订阅方案作为重度用户的最佳选择

### 新用户赠送

- 注册即送 **15 点**（可体验 15 次生成）
- 足够用户感受产品价值

## 五、订阅与点数兼容方案 ✅ 已确认

### 5.1 核心逻辑

```
用户发起生成请求
       ↓
   有活跃订阅？ ──是──→ 无限生成，不扣点
       │
       否
       ↓
   点数余额 ≥ 1？ ──否──→ 提示购买点数或订阅
       │
       是
       ↓
   扣除 1 点，生成内容
```

### 5.2 用户状态组合

| 状态 | 订阅 | 点数 | 行为 |
|------|------|------|------|
| A | ✅ 活跃 | 有/无 | 无限生成，不扣点 |
| B | ❌ 无/过期 | >0 | 每次扣 1 点 |
| C | ❌ 无/过期 | 0 | 引导购买 |

### 5.3 订阅取消处理

- 订阅取消后，当前周期结束前仍享受无限生成
- 周期结束后，如有剩余点数，转为点数模式
- 点数永久保留，不受订阅状态影响

### 5.5 数据库设计

```sql
-- 用户表扩展
ALTER TABLE users ADD COLUMN credits_balance INTEGER DEFAULT 0;

-- 点数购买记录
CREATE TABLE credit_purchases (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  package_id TEXT NOT NULL,
  points INTEGER NOT NULL,
  price DECIMAL(10,2) NOT NULL,
  paypal_order_id TEXT,
  purchased_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 点数消耗记录
CREATE TABLE credit_transactions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  type TEXT NOT NULL, -- 'purchase', 'consume', 'gift'
  points INTEGER NOT NULL, -- 正数为增加，负数为减少
  balance_after INTEGER NOT NULL,
  related_id TEXT, -- 关联的生成记录或购买记录
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 数据库设计补充

```sql
-- 生成记录表
CREATE TABLE generations (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  template_id TEXT NOT NULL,
  template_name TEXT NOT NULL,

  -- 输入输出
  input_data JSON NOT NULL,        -- 用户填写的表单数据
  output_content TEXT NOT NULL,    -- 生成的结果

  -- 消费信息
  credits_used INTEGER DEFAULT 0,  -- 消耗点数 (0=订阅用户)
  cost_type TEXT NOT NULL,         -- 'subscription' | 'credits'

  -- 元数据
  model_used TEXT,                 -- 使用的 AI 模型
  tokens_input INTEGER,            -- 输入 token 数
  tokens_output INTEGER,           -- 输出 token 数
  generation_time_ms INTEGER,      -- 生成耗时

  status TEXT DEFAULT 'success',   -- success, failed
  error_message TEXT,

  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,

  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 创建索引便于查询
CREATE INDEX idx_generations_user ON generations(user_id, created_at DESC);
CREATE INDEX idx_generations_template ON generations(template_id);
```
